Para este bloque de actividades vamos a repasar las peticiones HTTP GET a API REST públicas y la implementación de formularios estáticos. Para ello, primeramente deberás crear un componente header y tres vistas: enunciado, ejercicio1 y ejercicio2.

Para cada uno de los dos ejercicios siguientes deberás implementar los servicios, eventos e interfaces que consideres oportunos.

Ejercicio 1

Implementa el formulario proporcionado de manera que se permita recoger los datos introducidos por el usuario y, a partir de ellos, realizar una petición HTTP GET a la API REST pública de Rick and Morty. De este modo, en una primera instancia el usuario podrá seleccionar si quiere buscar un personaje o un capítulo. Posteriormente y dependiendo de la selección, se le permitirá introducir un nombre de un personaje o el número de un capítulo. Con el nombre del personaje se mostrará un listado de todos los personajes de la serie que contienen ese término de búsqueda, renderizándose en la interfaz el nombre y foto de cada uno de ellos. Con el nombre del capítulo introducido se mostrará el resultado de la petición con el número y nombre del capítulo (ejemplo: S01E02 - Lawnmower Dog) y un listado con el nombre y foto de todos los personajes que aparecen en dicho capítulo.

Ejercicio 2

Implementa el formulario proporcionado de manera que se permita recoger los datos introducidos por el usuario y, a partir de ellos, realizar una petición HTTP GET a la API REST pública de Ball Don't Lie! De este modo, en una primera instancia el usuario introducirá el nombre de un jugador de la NBA. A partir de este nombre, se ocultará el formulario y se mostrarán por pantalla todos los jugadores cuyo nombre o apellido coincide con el término de búsqueda. Si el usuario hace click sobre uno de esos jugadores, se ocultará el listado de jugadores y a continuación se mostrará el número de puntos totales encestados por ese jugador a lo largo de toda su carrera deportiva.
